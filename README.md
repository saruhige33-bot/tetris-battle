# Tetris Battle

CPU対戦と、友達とのオンライン対戦（ルームコード方式）に対応したテトリス対戦Webアプリ。攻撃要素あり（テトリス99風）— ライン消去で相手にお邪魔ブロック（ガベージライン）を送る。

## ローカルでの起動

```bash
npm install
npm start
```

`http://localhost:3000` をブラウザで開く。

## 操作方法

| 操作 | キー |
|---|---|
| 左移動 / 右移動 | ←　/　→ |
| ソフトドロップ | ↓ |
| ハードドロップ | Space |
| 右回転 | ↑ または X |
| 左回転 | Z |
| ホールド | C または Shift |

## 攻撃ルール

| 消去ライン数 | 送付ライン数 |
|---|---|
| シングル | 0 |
| ダブル | 1 |
| トリプル | 2 |
| テトリス（4ライン） | 4 |

コンボを継続するとボーナスが加算される。受け取った攻撃は即時挿入されず保留され、自分の攻撃と相殺後、相殺できなかった分が次の自分のミノ固定時に盤面へ挿入される。

## オンライン対戦（ルームコード方式）

1. 片方が「友達と対戦」→「ルームを作る」でコードを発行
2. もう片方がそのコードを「ルームコード入力」欄に入れて入室
3. 両者がそろうと自動的に対戦開始

## デプロイ（Render, 無料）

1. このリポジトリをGitHubにpush
2. [Render](https://render.com) で `New +` → `Web Service` → 対象リポジトリを接続
3. 設定:
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Instance Type: Free
4. デプロイ完了後に発行されるURL（`https://xxxx.onrender.com`）が常時アクセス可能なURLになる

**注意（Free Tierの制約）**: 15分間アクセスがないとサーバーがスリープし、次回アクセス時に起動まで数十秒かかる。

## 現状の制約

- オンライン対戦はクライアント権威モデル（各クライアントが自分の盤面を計算しサーバーは中継のみ）を採用しているため、改造クライアントによる不正は防げない。身内で遊ぶ前提のMVP。
- 7-bagのミノ順はルーム作成時にサーバーが配布するシードに基づき両者で同期される。
